## Rule Categories

- **Security Vulnerabilities**: Rules for detecting security issues like XSS, CSRF, and more
- **Performance Issues**: Rules for identifying performance bottlenecks
- **Code Quality**: Rules for ensuring code maintainability and readability
- **Best Practices**: Rules for enforcing Java best practices and conventions

## How to Use

To run these rules against your Java codebase:

```bash
code-pathfinder scan --lang java --rules java/security /path/to/your/project
```

For more information on using Code PathFinder with Java, see our [Java documentation](/docs/java).
