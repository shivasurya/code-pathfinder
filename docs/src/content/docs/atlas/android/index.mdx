        <div class="rule-header">
          <Icon name="warning" size="1.5rem" />
          <h3>WebView Misconfiguration</h3>
          <span class="severity high">High Severity</span>
        </div>
        <p>Detects insecure WebView configurations that could lead to JavaScript injection.</p>
        <div class="code-block">
          <pre><code class="language-java">WebView webView = new WebView(context);
WebSettings settings = webView.getSettings();
settings.setJavaScriptEnabled(true); // Potentially insecure</code></pre>
        </div>
      </div>

      <div class="rule-card">
        <div class="rule-header">
          <Icon name="warning" size="1.5rem" />
          <h3>Insecure File Access</h3>
          <span class="severity high">High Severity</span>
        </div>
        <p>Identifies insecure file access in Android applications.</p>
        <div class="code-block">
          <pre><code class="language-java">File file = new File(getExternalStorageDirectory(), "sensitive_data.txt");
FileOutputStream fos = new FileOutputStream(file); // Potentially insecure</code></pre>
        </div>
      </div>

      <div class="rule-card">
        <div class="rule-header">
          <Icon name="warning" size="1.5rem" />
          <h3>Implicit Intents</h3>
          <span class="severity medium">Medium Severity</span>
        </div>
        <p>Detects potentially insecure implicit intents.</p>
        <div class="code-block">
          <pre><code class="language-java">Intent intent = new Intent(Intent.ACTION_VIEW);
intent.setData(Uri.parse(userControlledUrl)); // Potentially insecure
startActivity(intent);</code></pre>
        </div>
      </div>
    </div>
  </div>

  <div class="usage-section">

    <div class="code-block command">

    </div>
    <p>For more information, see our <a href="/docs/android">Android documentation</a>.</p>
